# Утилита генерации тестовых данных (TestDataGenerator)

## Назначение

Утилита предназначена для генерации тестовых данных в формате TSV, имитирующих выгрузку банка. Используется для:

- Тестирования функционала импорта Financial Audit
- Наполнения базы данных тестовыми платежами
- Проверки работы REGEX-выражений при импорте
- Нагрузочного тестирования

## Сборка

Утилита собирается автоматически вместе с основным проектом:

```bash
cd build
cmake -G "MinGW Makefiles" ..
cmake --build .
```

Исполняемый файл: `build/TestDataGenerator.exe`

## Использование

### Синтаксис командной строки

```
TestDataGenerator <количество_записей> [имя_файла] [seed]
```

### Аргументы

| Аргумент | Обязательный | Описание |
|----------|--------------|----------|
| `количество_записей` | Да | Число записей для генерации (от 1 до 100 000) |
| `имя_файла` | Нет | Имя выходного TSV файла (по умолчанию: `bank_export.tsv`) |
| `seed` | Нет | Seed генератора случайных чисел для воспроизводимости |

### Примеры использования

```bash
# Сгенерировать 1000 записей в файл по умолчанию
TestDataGenerator 1000

# Сгенерировать 5000 записей в указанный файл
TestDataGenerator 5000 my_test_data.tsv

# Сгенерировать 10000 записей с фиксированным seed
TestDataGenerator 10000 data.tsv 42

# Показать справку
TestDataGenerator --help
```

## Формат выходного файла

### Структура TSV

Файл содержит следующие колонки (разделитель - табуляция):

| Колонка | Тип | Описание | Пример |
|---------|-----|----------|--------|
| `doc_number` | TEXT | Номер платежного поручения (6 цифр) | `000001` |
| `date` | TEXT | Дата платежа в формате ДД.ММ.ГГГГ | `15.01.2024` |
| `recipient` | TEXT | Наименование контрагента | `ООО "ТехноСервис"` |
| `inn` | TEXT | ИНН контрагента (10 цифр) | `7701234567` |
| `amount` | REAL | Сумма платежа в рублях | `125000.50` |
| `description` | TEXT | Назначение платежа | `Оплата по договору...` |

### Пример файла

```tsv
doc_number	date	recipient	inn	amount	description
000001	15.01.2024	ООО "ТехноСервис"	7701234567	125000.50	Оплата по договору №Д-1234 от 10.01.2024. Основание: акт №00567 от 12.01.2024. КОСГУ К262
000002	16.01.2024	АО "СтройМаш"	7702345678	350000.00	Авансовый платеж по контракту №К-5678 от 05.01.2024. К210
000003	17.01.2024	ИП Петров А.В.	770456789012	45000.00	Оплата услуг. накладная №12345 от 15.01.2024. К220
```

## Содержание описания платежа

Поле `description` генерируется автоматически и содержит информацию для тестирования импорта:

### Элементы описания

1. **Тип операции** (случайный выбор из списка):
   - Оплата по договору
   - Авансовый платеж
   - Окончательный расчет
   - Оплата услуг/товаров/работ
   - Возврат средств
   - Перечисление налога
   - Оплата аренды/связи/коммунальных услуг и т.д.

2. **Данные договора** (контракта):
   - Форматы: `по контракту №XXX`, `по договору №XXX`, `Контракт XXX`
   - Префиксы: К, Д, ГК, ДК, КД, Контракт
   - Пример: `по контракту №Д-1234 от 15.01.2024`

3. **Данные накладной/акта**:
   - Типы: акт, накладная, счет-фактура, УПД, акт выполненных работ
   - Пример: `акт №00567 от 12.01.2024`

4. **Код КОСГУ**:
   - Формат: `КXXX` (например, `К100`, `К262`, `К340`)
   - Диапазоны: 100-730 (доходы, расходы, активы, обязательства)

### Шаблоны описания

Утилита использует 4 шаблона для разнообразия:

1. **Шаблон 1**: Договор + КОСГУ
   ```
   Оплата по договору №Д-1234 от 15.01.2024. К262
   ```

2. **Шаблон 2**: Накладная + Договор + КОСГУ
   ```
   Оплата товаров. накладная №12345 от 10.01.2024. Контракт №Д-1234 от 05.01.2024. К210
   ```

3. **Шаблон 3**: Акт + КОСГУ
   ```
   акт №00567 от 12.01.2024. Оплата услуг. К220
   ```

4. **Шаблон 4**: Полный (операция + договор + накладная + КОСГУ)
   ```
   Оплата по договору №Д-1234 от 15.01.2024. Основание: акт №00567 от 12.01.2024. КОСГУ К262
   ```

## Генерируемые данные

### Контрагенты

Утилита использует предопределённый список из 30 контрагентов:
- ООО, АО, ЗАО, ИП
- Различные сферы деятельности
- Реалистичные ИНН

### Диапазоны данных

| Параметр | Минимум | Максимум |
|----------|---------|----------|
| Сумма платежа | 1 000 ₽ | 500 000 ₽ |
| Год договора | 2022 | 2024 |
| Год платежа | 2023 | 2025 |
| Номер договора | 1 | 9999 |
| Номер накладной | 1 | 99999 |

## Производительность

| Количество записей | Время генерации | Размер файла |
|--------------------|-----------------|--------------|
| 1 000 | ~0.01 сек | ~0.2 МБ |
| 10 000 | ~0.03 сек | ~1.5 МБ |
| 100 000 | ~0.3 сек | ~15 МБ |

*Время указано ориентировочно для современного ПК.*

## Воспроизводимость результатов

Для получения одинаковых результатов при разных запусках используйте параметр `seed`:

```bash
# Оба вызова сгенерируют одинаковые файлы
TestDataGenerator 1000 test1.tsv 42
TestDataGenerator 1000 test2.tsv 42
```

## Интеграция с Financial Audit

### Импорт сгенерированных данных

1. Запустите Financial Audit
2. Откройте или создайте базу данных
3. Перейдите в меню **Сервис → Импорт из TSV**
4. Выберите сгенерированный файл
5. Настройте маппинг колонок:
   - `date` → Дата
   - `doc_number` → Номер документа
   - `recipient` → Получатель
   - `inn` → ИНН (если есть такая колонка)
   - `amount` → Сумма
   - `description` → Описание
6. Настройте REGEX (должны совпадать с форматами в описании):
   - Договоры: `(?:по контракту|по контр|Контракт|дог\.|К-т)(?:№)?\s*([^\s,]+)\s*(?:от\s*)?(\d{2}\.\d{2}\.(?:\d{4}|\d{2}))`
   - КОСГУ: `К(\d{3})`
   - Накладные: `(?:акт|сч\.?|сч-ф|счет на оплату|№)\s*([^\s,]+)\s*(?:от\s*)?(\d{2}\.\d{2}\.(?:\d{4}|\d{2}))`
7. Нажмите **Начать импорт**

### Проверка результатов

После импорта проверьте:
- Созданные платежи в форме **Банк (Платежи)**
- Созданные записи КОСГУ в форме **КОСГУ**
- Созданные договоры в форме **Договоры**
- Созданные контрагенты в форме **Контрагенты**
- Созданные накладные в форме **Накладные**

## Ограничения

- Максимальное количество записей: 100 000
- Кодировка файла: UTF-8
- Разделитель колонок: табуляция (TAB)
- Разделитель строк: перевод строки (LF/CRLF)
