# Техническое задание на разработку приложения "Financial Audit"
## Система для финансового аудита организаций государственного сектора

---

## 1. Общие сведения

### 1.1. Наименование проекта
**Financial Audit** — настольное приложение для финансового аудита организаций государственного сектора.

### 1.2. Заказчик и разработчик
- **Заказчик:** GorINIch perconal
- **Разработчик:** GorINIch & Co

### 1.3. Основание для разработки
Автоматизация процессов финансового аудита целевого использования средств, включая учёт платежей, контрагентов, договоров (закупок), накладных и классификатора КОСГУ.

---

## 2. Назначение и цели разработки

### 2.1. Назначение системы
Приложение предназначено для:
- Ведения базы данных финансовых операций организации в разрезе статей затрат для анализа обоснованности затрат
- Управления справочниками (КОСГУ, контрагенты, договоры, накладные) с возможностью фильтации и групповыпи операциями
- Импорта данных из внешних источников (TSV-файлы) с автоматическим заполнением справочников по назначению платеда и контрагенту
- Экспорта данных и формирования отчётности (PDF, CSV)
- Выполнения SQL-запросов к базе данных в форму анализа и экспорта для анализа состояния учета, правомерности закупок с составлением отчетов и сводов
- Выявления подозрительных операций на основе настраиваемых правил

### 2.2. Цели разработки
- Повышение эффективности работы аудиторов
- Снижение количества ручных операций при вводе данных
- Автоматизация проверки финансовой документации
- Развитая фильтрация и отчетность для эффекктивного выявления нецелевых расходов
- Централизованное хранение и управление финансовыми данными

## 4. Структура базы данных

### 4.1. Схема БД

```sql
-- Справочник КОСГУ
CREATE TABLE KOSGU (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    code TEXT NOT NULL UNIQUE,
    name TEXT NOT NULL,
    note TEXT
);

-- Справочник контрагентов
CREATE TABLE Counterparties (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    inn TEXT UNIQUE,
    is_contract_optional INTEGER DEFAULT 0
);

-- Справочник договоров
CREATE TABLE Contracts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    number TEXT NOT NULL,
    date TEXT NOT NULL,
    counterparty_id INTEGER,
    contract_amount REAL DEFAULT 0.0,
    end_date TEXT,
    procurement_code TEXT,
    note TEXT,
    is_for_checking INTEGER DEFAULT 0,
    is_for_special_control INTEGER DEFAULT 0,
    is_found INTEGER DEFAULT 0,
    FOREIGN KEY(counterparty_id) REFERENCES Counterparties(id)
);

-- Справочник платежей
CREATE TABLE Payments (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    date TEXT NOT NULL,
    doc_number TEXT,
    type INTEGER NOT NULL DEFAULT 0,
    amount REAL NOT NULL,
    recipient TEXT,
    description TEXT,
    counterparty_id INTEGER,
    note TEXT,
    FOREIGN KEY(counterparty_id) REFERENCES Counterparties(id)
);

-- Справочник накладных
CREATE TABLE Invoices (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    number TEXT NOT NULL,
    date TEXT NOT NULL,
    contract_id INTEGER,
    total_amount REAL DEFAULT 0.0,
    FOREIGN KEY(contract_id) REFERENCES Contracts(id)
);

-- Детали платежей (связь платежей с КОСГУ, договорами, накладными)
CREATE TABLE PaymentDetails (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    payment_id INTEGER,
    kosgu_id INTEGER,
    contract_id INTEGER,
    invoice_id INTEGER,
    amount REAL,
    FOREIGN KEY(payment_id) REFERENCES Payments(id),
    FOREIGN KEY(kosgu_id) REFERENCES KOSGU(id),
    FOREIGN KEY(contract_id) REFERENCES Contracts(id),
    FOREIGN KEY(invoice_id) REFERENCES Invoices(id)
);

-- Регулярные выражения
CREATE TABLE Regexes (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    pattern TEXT NOT NULL
);

-- Подозрительные слова
CREATE TABLE SuspiciousWords (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    word TEXT NOT NULL
);

-- Настройки
CREATE TABLE Settings (
    id INTEGER PRIMARY KEY,
    organization_name TEXT,
    period_start_date TEXT,
    period_end_date TEXT,
    note TEXT,
    import_preview_lines INTEGER,
    theme INTEGER,
    font_size INTEGER
);
```

### 4.2. Файл недавних файлов

**Список последних открытых баз данных хранится в текстовом файле:**

| Параметр | Значение |
|----------|----------|
| **Имя файла** | `recent_files.txt` |
| **Расположение** | Директория приложения (или `%APPDATA%/FinancialAudit/` на Windows) |
| **Формат** | Одна строка на файл (полный путь к базе данных) |
| **Кодировка** | UTF-8 |
| **Максимум записей** | 10 |
| **Сортировка** | По дате открытия (последняя открытая — первая в списке) |

**Пример содержимого `recent_files.txt`:**
```
C:/Users/User/Documents/financial_audit.db
D:/Work/audits/2026/march.db
/home/user/databases/main.db
```

**Операции с файлом:**
- **Чтение:** При запуске приложения загружается список для меню "Файл → Недавние файлы"
- **Запись:** При открытии/создании базы путь добавляется в начало файла
- **Обновление:** Если файл уже есть в списке, он перемещается в начало
- **Очистка:** Старые записи (после 10-й) удаляются

```cpp
// Основной шрифт с кириллицей
```

### Приложение Б. Диалоги работы с файлами (ImGuiFileDialog)

**Библиотека ImGuiFileDialog используется для всех операций с файлами:**

| Операция | Диалог | Фильтры |
|----------|--------|---------|
| Создание БД | SaveDialog | `.db,.sqlite,.sqlite3` |
| Открытие БД | OpenFileDialog | `.db,.sqlite,.sqlite3` |
| Сохранение как | SaveDialog | `.db,.sqlite,.sqlite3` |
| Импорт TSV | OpenFileDialog | `.tsv,.csv,.txt` |
| Импорт CSV | OpenFileDialog | `.csv,.txt` |
| Экспорт CSV | SaveDialog | `.csv` |
| Экспорт PDF | SaveDialog | `.pdf` |

**Пример использования:**
```cpp
// Диалог открытия базы данных
ImGuiFileDialog::Instance()->OpenDialog("OpenDbDialog", "Открыть базу данных",
    "Database Files{.db,.sqlite,.sqlite3}", ".");

if (ImGuiFileDialog::Instance()->Display("OpenDbDialog")) {
    if (ImGuiFileDialog::Instance()->IsOk()) {
        std::string filePath = ImGuiFileDialog::Instance()->GetFilePathName();
        dbManager->openDatabase(filePath);
    }
    ImGuiFileDialog::Instance()->Close();
}

// Диалог сохранения базы
ImGuiFileDialog::Instance()->SaveDialog("SaveDbDialog", "Сохранить базу как...",
    "Database Files{.db,.sqlite,.sqlite3}", "financial_audit.db");

if (ImGuiFileDialog::Instance()->Display("SaveDbDialog")) {
    if (ImGuiFileDialog::Instance()->IsOk()) {
        std::string filePath = ImGuiFileDialog::Instance()->GetFilePathName();
        dbManager->backupTo(filePath);
    }
    ImGuiFileDialog::Instance()->Close();
}
```

